useradd, usermod, userdel

Benutzerkonten anlegen, ändern und entfernen – inklusive Home, Shell und Expiry.

LPIC-1 Themen

Grundlagen & Philosophien Dateisysteme & FHS Berechtigungen & ACLs Paketverwaltung Prozesse & Systemd Benutzer & Gruppen Netzwerkgrundlagen Shells & Scripting Archivierung & Komprimierung Systemstart & Boot Hardware & Geräte

Unterthemen

useradd, usermod, userdel passwd, shadow, chage groups & gpasswd sudoers & visudo PAM-Grundlagen

Lernziele

  • Benutzer korrekt anlegen (Home, Shell, UID/GID, Skel).
  • Parameter ändern (Gruppen, Ablaufdaten, Sperren).
  • Konten sauber entfernen oder deaktivieren.

Grundlagen

# Anlegen
sudo useradd -m -s /bin/bash alice
sudo passwd alice

# Ändern
sudo usermod -aG wheel,dev alice
sudo usermod -L alice           # sperren (-U entsperren)
sudo chage -E 2025-12-31 alice  # Expiry

# Löschen
sudo userdel alice              # Konto
sudo userdel -r alice           # inkl. Home

Systemkonten (UID < 1000) werden meist ohne Login-Shell und Home angelegt. Standards definiert /etc/login.defs und das Skel unter /etc/skel.

Cheatsheet

getent passwd alice
getent group dev
id alice

Übungen

  1. Lege analyst mit ablaufendem Passwort an und füge es zu audit hinzu.
  2. Sperre temporär das Konto guest und prüfe den Status.

Zu meinem Fortschritt

Vertiefung & Praxis

Vertiefung: Benutzer- und Gruppenverwaltung.

  • Benutzer-Lebenszyklus: anlegen → Passwort setzen → Gruppenmitgliedschaften → Shell/Home → Ablaufdatum → Sperren/Entsperren.
  • Dateirechte & Ownership im Zusammenspiel mit primärer und sekundären Gruppen.
  • PAM-Module (Pluggable Authentication Modules): Reihenfolge und Kontrollflags (required, requisite, sufficient, optional).

Kommandoreferenz

# Benutzer anlegen (mit Home, Standardgruppe, Shell)
sudo useradd -m -s /bin/bash -U alice
sudo passwd alice

# Benutzer ändern (Shell, Gruppen, Ablauf)
sudo usermod -s /bin/zsh -aG wheel,audio alice
sudo chage -E 2025-12-31 alice

# Benutzer sperren/entsperren
sudo usermod -L alice   # sperren
sudo usermod -U alice   # entsperren

# Benutzer löschen (+Home)
sudo userdel -r alice

# Gruppenverwaltung
getent group developers
sudo groupadd developers
sudo gpasswd -a alice developers
sudo gpasswd -d alice developers

Praxis-Lab

  1. Lege proj‑Gruppe an, füge 2 Nutzer hinzu.
  2. Erstelle /srv/proj und setze sgid, damit neue Dateien Gruppen‑Ownership erben.
  3. Erzwinge sichere Passwörter via pam_pwquality.so (minlen, ucredit/lcredit/dcredit).

Häufige Fehler

  • Permission denied trotz Mitgliedschaft → neu einloggen (Gruppen gelten erst für neue Sessions).
  • nologin-Shell gesetzt → interaktive Anmeldung scheitert.
  • Home nicht erstellt: useradd -m vergessen.

Prüfungstipps

  • Unterschied primäre vs. sekundäre Gruppe kennen.
  • chage‑Felder: minimale/maximale Tage, Warnzeit, Inaktivität.
  • visudo verwenden, niemals /etc/sudoers direkt editieren.

Checkliste – schneller Durchblick

  • Relevante Logdateien geprüft? (journalctl, /var/log/*)
  • Manpage/--help gelesen und Option bewusst gewählt?
  • Rechte/Ownership/SELinux/AppArmor berücksichtigt?
  • Distribution & Version: gleiches Verhalten verifiziert?

Optionen‑Atlas (Kernbefehle)

KommandoWichtige Optionen
useradd-m Home anlegen; -s Shell; -U Gruppe erstellen; -G Sekundärgruppen; -e Ablaufdatum (YYYY-MM-DD)
usermod-aG Gruppen anhängen; -L sperren; -U entsperren; -s Shell; -e Ablaufdatum; -c Kommentar
userdel-r Home & Mailspool löschen; (Achtung: Dateien auf anderen Mounts bleiben!)
chage-l anzeigen; -E Ablaufdatum; -m minimal; -M maximal; -W Warnzeit; -I Inaktivität
passwd-l sperren; -u entsperren; -e sofort ablaufen lassen; -n minimal; -x maximal; -w Warnzeit
gpasswd-a hinzufügen; -d entfernen; -A Admins; -M Mitglieder setzen

Szenarien aus der Praxis

  • Projektordner /srv/proj mit sgid: neue Dateien erben Gruppen‑Ownership.
  • Service-Account ohne Login: Shell /usr/sbin/nologin und keine Home-Erstellung.
  • Passwortpolitik mit pam_pwquality: Mindestlänge & Zeichencocktail erzwingen.
  • Temporäre Adminrechte via sudoers Drop-In in /etc/sudoers.d/NAME.

Diagnose-Flow (schneller Pfad zur Lösung)

  1. Existiert der Nutzer/Gruppe? (getent passwd/group)
  2. Richtige Shell/Home gesetzt? (useradd/usermod -s/-m)
  3. Passwort/Lock‑Status ok? (passwd -S, chage -l)
  4. Gruppenmitgliedschaften aktiv? (id; neue Session nötig)
