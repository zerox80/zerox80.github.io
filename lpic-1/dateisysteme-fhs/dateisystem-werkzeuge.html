Cheatsheet (Kurz)

# Erstellen
mkfs.ext4 /dev/sdb1
mkfs.xfs  /dev/sdb2
mkfs.btrfs /dev/sdb3

# Prüfen/Reparieren (offline!)
e2fsck -f /dev/sdb1
xfs_repair /dev/sdb2
btrfs check --readonly /dev/sdb3

# Infos/Tuning
tune2fs -l /dev/sdb1
xfs_info /mnt
btrfs filesystem df /mnt

mkfs im Detail

  • ext4: mkfs.ext4 -L data -m 0 -O metadata_csum,64bit /dev/sdb1 (Label, Reserve für root=0%, moderne Features).
  • XFS: mkfs.xfs -L media -m crc=1 -i size=512 /dev/sdb2 (CRC‑Metadaten, größere Inodes für xattrs).
  • Btrfs: mkfs.btrfs -L bdata -m dup -d single /dev/sdb3 (Metadaten‑Redundanz auf Single‑Disk).
# Nach Erstellung: prüfen
lsblk -f
blkid

Loopback‑Lab (sicher testen)

# 1) Sparse‑Datei anlegen
truncate -s 2G /tmp/disk.img

# 2) Loop‑Device zuordnen
LOOP=$(sudo losetup -fP --show /tmp/disk.img)
echo $LOOP  # z. B. /dev/loop5

# 3) Partitionstabelle + Partition (optional)
sudo parted -s $LOOP mklabel gpt mkpart p1 ext4 1MiB 100%
sudo partprobe $LOOP

# 4) Dateisystem erstellen & mounten
sudo mkfs.ext4 ${LOOP}p1
sudo mkdir -p /mnt/testfs && sudo mount ${LOOP}p1 /mnt/testfs

# 5) Arbeiten/Tests durchführen, dann aufräumen
sudo umount /mnt/testfs
sudo losetup -d $LOOP

fsck / Repair‑Tools

  • ext4: e2fsck (fsck.ext4). Nur unmounted nutzen; -f erzwingt Check; -y auto‑fix mit Vorsicht.
  • XFS: Kein fsck.xfs (Wrapper). Verwende xfs_repair; erst -n (Read‑only) zur Analyse, dann ohne -n.
  • Btrfs: btrfs check vorsichtig, standardmäßig --readonly; bevorzugt laufende Pflege mit btrfs scrub.
# ext4 Zwangscheck konfigurieren
sudo tune2fs -c 30 -i 6m /dev/sdb1   # alle 30 Mounts oder 6 Monate

# XFS Read‑only Analyse
sudo xfs_repair -n /dev/sdb2

# Btrfs Scrub (on‑line)
sudo btrfs scrub start -B /mnt

Tuning & Admin‑Tools

  • ext4: tune2fs (Features, Reserve, Checks), dumpe2fs (Metadaten), debugfs (Experten).
  • XFS: xfs_admin (Label/UUID), xfs_growfs, xfs_spaceman, xfs_fsr (Defrag).
  • Btrfs: btrfs subvolume/create/snapshot/send/receive/balance/filesystem df/du/device stats.
# Labels/UUID
sudo e2label /dev/sdb1 DATA
sudo xfs_admin -L MEDIA /dev/sdb2
sudo btrfs filesystem label /mnt MEDIA

# Grow (online, wenn darunter LVM/Device größer)
sudo resize2fs /dev/sdb1
sudo xfs_growfs /mnt

Imaging & Wipe

  • Imaging: dd, ddrescue, partclone (FS‑aware), fsarchiver.
  • Wipe/Signaturen: wipefs zum Entfernen alter FS‑Signaturen; blkdiscard für SSD‑Trim.
# Vorsicht: destruktiv!
sudo wipefs -a /dev/sdb
sudo dd if=/dev/zero of=/dev/sdb bs=1M status=progress  # nur wenn du sicher bist

Fehlersuche

  • Gerätepfade ändern sich: immer UUID/LABEL in fstab nutzen.
  • Busy Mounts: lsof +f -- /mnt, fuser -vm /mnt, dann Prozesse beenden.
  • IO‑Fehler: dmesg/journal prüfen, SMART (smartctl -a), Kabel/Ports.
