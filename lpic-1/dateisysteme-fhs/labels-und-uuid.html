Labels & UUID

Unterthemen

FHS Überblick Mount & /etc/fstab Dateisystem‑Werkzeuge Dateisystem‑Typen Speicheranalyse Inodes & Links Quotas Partitionen & Swap Labels & UUID Mount‑Optionen & Sicherheit Dateisystem‑Attribute tmpfs & Bind‑Mounts

Lernziele

  • UUIDs/Labels auslesen und setzen.
  • fstab‑Einträge mit UUID/Label pflegen.

Cheatsheet

blkid
e2label /dev/sdb1 DATA
tune2fs -L DATA /dev/sdb1
# fstab mit LABEL/UUID
UUID=xxxx-xxxx /data ext4 defaults 0 2
LABEL=DATA   /data ext4 defaults 0 2

Theorie

  • UUID: Eindeutiger Bezeichner eines Dateisystems oder einer Partition. Stabil gegenüber wechselnden /dev/sdX-Namen.
  • Label: Menschlich lesbarer Name. Kann kollidieren, sollte eindeutig innerhalb des Systems sein.
  • Wo gespeichert? In der Superblock/Metadaten des Dateisystems (ext*, XFS, Btrfs) oder Partition (GPT PARTUUID/PARTLABEL).
  • fstab-Auflösung: Kernel/systemd lösen UUID= und LABEL= via udev/blkid auf.
  • Partition vs. FS: Unterscheide PARTUUID/PARTLABEL (Partition) von UUID/LABEL (Dateisystem).

Praxis

# Alle Bezeichner anzeigen
lsblk -f
blkid

# Label setzen/ändern
e2label /dev/sdb1 DATA         # ext2/3/4
tune2fs -L DATA /dev/sdb1      # Alternative für ext*
xfs_admin -L MEDIA /dev/sdb2   # XFS
btrfs filesystem label /mnt BDATA  # Btrfs (online auf Mountpoint)

# UUID anzeigen/ändern (nur wenn nötig!)
tune2fs -l /dev/sdb1 | grep UUID
# Achtung: UUID ändern kann Boot brechen, nur mit Plan ändern:
# tune2fs -U random /dev/sdb1

# fstab Einträge mit UUID/LABEL
UUID=xxxx-xxxx  /data  ext4  noatime  0 2
LABEL=MEDIA     /media xfs   defaults 0 0

# GPT Partition Label/UUID
parted /dev/sdb name 1 DATA
blkid -o export /dev/sdb1 | egrep 'PARTLABEL|PARTUUID'

Best Practices

  • UUID bevorzugen: Maximale Stabilität, insbesondere bei wechselnder Geräte-Reihenfolge.
  • Eindeutige Labels: Kurz, sprechend, ohne Leerzeichen; für Menschen hilfreich (DATA, BACKUP, MEDIA).
  • Konsistenz: Einheitliche Strategie (UUID oder LABEL) für alle Mounts dokumentieren.
  • Änderungen planen: UUID/Label-Änderungen immer mit Anpassung von /etc/fstab, Crypttab, Bootloader.

Troubleshooting

  • Mount scheitert nach Hardwarewechsel: Prüfe mit blkid/ lsblk -f, ob UUID/Label noch stimmt; fstab korrigieren.
  • Doppelte Labels: blkid -L NAME zeigt erstes Matching; Labels eindeutig vergeben.
  • Alte Signaturen: Vor Neuformatierung wipefs -a /dev/sdX nutzen, um Mehrdeutigkeiten zu vermeiden.
  • Initramfs Cache: Nach wichtigen Änderungen ggf. update-initramfs -u bzw. dracut -f.

Erweitertes Cheatsheet

# Auflösen
blkid -L DATA
blkid -U 1234-ABCD

# Exportformat praktisch fürs Skripting
blkid -o export /dev/sdb1

# Udev-Properties ansehen
udevadm info --query=all --name=/dev/sdb1 | less

# Systemd: Automount mit UUID
UUID=xxxx-xxxx /data ext4 noatime,x-systemd.automount 0 2

Übungen

  1. Vergib eindeutige Labels für 2 Partitionen und mounte sie per Label in /etc/fstab.
  2. Ersetze in einem Testsystem alle /dev/sdX-Einträge in fstab durch UUID= und validiere mit mount -a.
  3. Zeige PARTLABEL/PARTUUID einer GPT-Partition und erkläre den Unterschied zur FS-UUID.
