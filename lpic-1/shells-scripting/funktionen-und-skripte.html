Beispielstruktur

#!/usr/bin/env bash
set -Eeuo pipefail

usage() { echo "Usage: $0 -n NAME"; }

log()   { printf '[%s] %s\n' "$(date +%T)" "$*"; }
err()   { echo "ERROR: $*" >&2; }

greet() {
  local name=${1:?name required}
  echo "Hallo $name"
}

main() {
  local name=""
  while getopts ':n:h' opt; do
    case $opt in
      n) name=$OPTARG ;;
      h) usage; exit 0 ;;
      *) usage; exit 2 ;;
    esac
  done
  [[ -n $name ]] || { usage; exit 2; }
  greet "$name"
}

main "$@"

Trenne Hilfsfunktionen (Logging, Fehler) von main. Nutze getopts für robuste Optionsverarbeitung.

Vertiefung & Praxis

Vertiefung: Shell-Workflows & Scripting.

#!/usr/bin/env bash
set -Eeuo pipefail
IFS=$'\n\t'

# Funktionen & Fehlerbehandlung
log() { printf '[%s] %s\n' "$(date +%T)" "$*"; }
trap 'echo "Fehler in Zeile $LINENO" >&2' ERR

# Parameter & Defaults
name=${1:-world}
log "Hallo $name"

# Pipes & Prozesssubstitution
awk -F: '{print $1}' /etc/passwd | sort | uniq
comm -13 <(cut -d: -f1 alt.txt | sort) <(cut -d: -f1 neu.txt | sort)

# grep/sed/awk Patterns
grep -E '^(UID|GID)=' /etc/login.defs
sed -n '1,10p' datei
awk '$3 >= 1000 {print $1,$3}' FS=: OFS='\t' /etc/passwd

Best Practices

  • shellcheck gegen häufige Fehler nutzen.
  • Niemals ungequotete Variablen in Globs/Word-Splitting ("$var").
  • Klare Exit-Codes und set -e bewusst einsetzen.

Checkliste – schneller Durchblick

  • Relevante Logdateien geprüft? (journalctl, /var/log/*)
  • Manpage/--help gelesen und Option bewusst gewählt?
  • Rechte/Ownership/SELinux/AppArmor berücksichtigt?
  • Distribution & Version: gleiches Verhalten verifiziert?
