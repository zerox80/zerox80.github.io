Grundregeln

  • Nutze #!/usr/bin/env bash und set -Eeuo pipefail am Anfang.
  • Quote Variablen grundsätzlich: "$var". Nutze Arrays statt $*.
  • Behandle Fehler zentral (Trap auf ERR) und logge mit Zeitstempel.
  • Trenne Logik in Funktionen (main, parse_args, run).
  • Schreibe Hilfe/Usage und prüfe Eingaben strikt.

Templates

#!/usr/bin/env bash
set -Eeuo pipefail
shopt -s nullglob dotglob
trap 'rc=$?; echo "[ERR] ($rc) in Zeile $LINENO" >&2' ERR

usage(){ cat <<USAGE
Usage: $0 [-v] -i INPUT -o OUTPUT
USAGE
}

parse_args(){ local OPTIND; while getopts ':vi:o:h' o; do case $o in
  v) VERBOSE=1;; i) IN=$OPTARG;; o) OUT=$OPTARG;; h) usage; exit 0;; *) usage; exit 2;; esac; done }

main(){ :; }
main "$@"

Übungen

  1. Baue ein Projektskelett (Ordner, Skripte, README) und lege ein Skript-Template an.
